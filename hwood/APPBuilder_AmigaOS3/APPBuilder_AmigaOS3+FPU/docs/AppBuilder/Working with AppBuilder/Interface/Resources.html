Resources

 

Your application may need resource files like images, sounds, databases, that are needed at runtime, data that you have to provide in your distribution packages.

For example, suppose you are developing a game that needs the following resources:
C:/Development/MyGame/Graphics/
C:/Development/MyGame/Sounds/
C:/Development/MyGame/Maps/
C:/Development/MyGame/levels.dat

 

With a manual build process you have to copy the folders Graphics, Sounds and Maps, along with the file levels.dat where your executable has been created. This must be done for every platform you wish to distribute for.

This section can help you a lot to accomplish this task because you have to tell AppBuilder what to include in your distribution and at build time the program will take care of copying the contents in the right position, including for MacOS application boundles.

As you can see, at the bottom of this area, you have three buttons you can use to add files, to add folders and to remove entries into/from the resources list.
