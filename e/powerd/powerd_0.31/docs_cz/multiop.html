• popis: tyto jsou podobne jedno znamenkum, ale lze je pouzit i pro vice promennych najednou: 
      NEG a,b,c

    je to same jako: 
      a:=-a
  b:=-b
  c:=-c
  • syntaxe: 
      NEG a   // je to same jako a:=-a
  NOT a   // je to same jako a:=~a
  INC a   // je to same jako ++a
  DEC a   // je to same jako --a
  • popis NEW:
    NEW vola funkci pro alokaci pameti, velikost je dana viz nize, a uklada ukazatel na tuto pamet do dane promenne. jestlize alokace pameti selze, je vyvolana vyjimka "MEM", kterou lze zachitit pomoci EXCEPT/EXCEPTDO. 
      DEF a:PTR TO obj,b=20,c=3,d:PTR TO DOUBLE

  NEW a
  // je to same jako IF (a:=AllocVec(SIZEOF_obj,MEMF_PUBLIC|MEMF_CLEAR))=NIL THEN Raise("MEM")

  NEW a[10]
  // je to same jako IF (a:=AllocVec(10*SIZEOF_obj,MEMF_PUBLIC|MEMF_CLEAR))=NIL THEN Raise("MEM")

  NEW a[b*c+2]
  // je to same jako IF (a:=AllocVec((b*c+2)*SIZEOF_obj,MEMF_PUBLIC|MEMF_CLEAR))=NIL THEN Raise("MEM")

  NEW d[10]
  // je to same jako IF (d:=AllocVec(10*SIZEOF_DOUBLE,MEMF_PUBLIC|MEMF_CLEAR))=NIL THEN Raise("MEM")

    lze take pouzit seznam alokaci jako: 
      NEW d[4],a[b]
  • popis END:
    END musi byt pouzito pro korektni dealokaci pameti alokovane pomoci NEW. 
      NEW d[4],a[b]
  ...
  END a,d
