    Pokud chcete pouzit vice jak dvou rozmerna pole, nelze to provest takto: 
        field:PTR TO PTR TO PTR TO PTR TO ...

    Je potreba to provest jednim z nasledujucich zpusobu

  • Vicerozmerna pole:
    Nasledujici dve ukazke jsou totozne: 
      DEF field[10,20]:LONG
  field[3,4]:=123

  DEF field[10*20]:LONG
  field[4*10+3]:=123


  • Vicerozmerna pole skrz ukazatele:
    Dve predchozi ukazke alokovaly 10*20*SIZEOF_LONG bajtu pameti, ale urcite je take potreba pouziti vicerozmernych poli bez nutne alokace pameti (ktera muze byt vytvorena nekde jinde). Staci pridat pred specifikaci daneho rozmeru pole dvojtecku: 
      DEF field[:10]:LONG
  ...
  alokace pameti pro promennou field
  ...
  field[3,4]:=123


  DEF field:PTR TO LONG
  ...
  alokace pameti pro promennou field
  ...
  field[4*10+3]:=123

    Tato definice nealokuje zadnou pamet, ale kompilator si ulozi rozmer daneho pole.
