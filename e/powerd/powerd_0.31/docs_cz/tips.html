• Uvod:
    PowerD ma velmi vsestrannou syntaxi, coz znamena, ze v mnoha pripadech lze uzit ruznych zkratek, ci zjednoduseni zdrojoveho kodu. Vetsina z techto ukazek je napsana v danych dokumentech, tento dokument slouzi k tomu, abyste je mohli videt vsechny pohromade a nemuseli je vyhledavat.

  • SELECT <konstanta>:
    Pokud chcete napriklad porovnavat promennou z ruznymi retezci, lze k tomu pouzit prikaz IF: 
      IF StrCmp(name,'one')
    x:=1
  ELSEIF StrCmp(name,'two')
    x:=2
  ELSEIF StrCmp(name,'three')
    x:=3
  ELSEIF StrCmp(name,'four')
    x:=4
  ELSE
    x:=0
  ENDIF

    Ale prikaz SELECT podporuje konstantu jako porovnavaci bazi, takze se da predchozi ukazka takto zjednodusit: 
      SELECT TRUE
  CASE StrCmp(name,'one');   x:=1
  CASE StrCmp(name,'two');   x:=2
  CASE StrCmp(name,'three'); x:=3
  CASE StrCmp(name,'four');  x:=4
  DEFAULT;                   x:=0
  ENDSELECT

  • Podminky:
    Podminky v PowerD vzdy vraceji bud TRUE, jeli podminka pravda, nebo FALSE, pokud ne. 
      a:=IF <condition> THEN TRUE ELSE FALSE

    je to same jako: 
      a:=<condition>

  • Vicenasobne podminky:
    PowerD (jako jediny programovaci jazyk?) podporuje vicenasobne podminky jako: 
      IF a>b>c THEN PrintF('True\n')

    Coz je vnitrne prevedeno na: 
      IF a>b AND b>c THEN PrintF('True\n')

  • Zkraceni prirazeni pomoci PreProcessoru:
    PowerD proprocessor kontrokuje kazde <var>:=.<item>: Kazde: 
      var:=.item

    bude prevedeno na: 
      var:=var.item

    Ale pozor, toto je rys pouze preprocesoru, takze jde pouze o zpracovani slov a znaku zdrojoveho kodu, preprocessor netestuje nic v tomto pripade.
