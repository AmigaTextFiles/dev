videopcm.decoder/background

DESCRIPTION 
   The class is designed to convert some formats of uncompressed pixmaps to
        one of Reggae common formats. Supported pixel formats are:

   MMF_VIDEO_GRAY8 - grayscale pixmap with 8-bit pixels.
   MMF_VIDEO_RGB24 - packed RGB format, 8 bits per component in red, green,

   No line padding is assumed.
NEW ATTRIBUTES 
   Attributes applicability:
     I - may be set at creation time.
     S - may be set on an existing object.
     G - may be get from an object.
     P - may be set for an object's port.
     Q - may be queried from an object's port.

   MMA_Video_Height         (V51) [..G.Q], ULONG
   MMA_Video_Width          (V51) [..G.Q], ULONG
   MMA_Video_UseAlpha       (V51) [.SGPQ], BOOL
NEW METHODS 
   MMM_Pull(port, buffer, length)  (V50)
HISTORY 
   51.2 (02.02.2006)
   - Added support for MMF_VIDEO_RGB24.

   51.1 (29.10.2006)
   - Full support for MMF_VIDEO_GRAY8.

videopcm.decoder/MMA_Video_UseAlpha

NAME 
   MMA_Video_UseAlpha (V51) [.SGPQ], BOOL
FUNCTION 
   Controls usage of alpha channel data provided on input. The default
   value for this attribute is FALSE. Then, when the object input is
   connected to a previous object, this previous object (a demuxer usually)
   is asked for alpha channel data presence. An application can override the
   attribute value later. For videopcm.decoder this attribute has meaning
   only for input formats providing alpha channel data (none at the moment).

videopcm.decoder/MMM_Pull

NAME 
   MMM_Pull -- Converts pixel data to a common format. (V50)
SYNOPSIS 
   ULONG DoMethod(Object *obj, MMM_Pull, ULONG port, APTR buffer, ULONG
   length);
FUNCTION 
   The method pulls pixel data from input and converts them to ARGB32
   pixels. ARGB32 values are placed in the passed buffer.
INPUTS 
   obj - object to perform method on.
   port - number of port, data must be pulled from output, so 1.
   buffer - destination buffer for common format image data. This buffer
     must be allocated with MediaAllocVec().
   length - amount of data to be pulled in bytes. It should be evenly
     divisible by number of bytes per pixel for the common format (4 for
     MMFC_VIDEO_ARGB32), if not, it is rounded down, and the rest of
     bytes are undefined.
RESULT 
   Number of bytes pulled, secondary error information via MMA_ErrorCode.
