                                  Sharing a game

1.Automatically
---------------
1-Open your project
2-Click on Build; a Build folder will be created with the necessary files
3-Select your destination platform

    Low Spec Amiga: When you target A500 or similar. Game will still run in 
                    all AMIGAs.
	Restrictions: 32 objects per level maximum. 
                      200 Frames maximum
                      No CD capabilities
                      No AGA parallax (ECS parallax allowed)
                      No custom OS fonts (bitmap fonts allowed)

    Generic: For All AMIGAs.

    68020: Code optimized for Motorola 68020 or higher.

4-Select if assets must be compressed, compressing will make them smaller in 
  disk.
5-Select if you want to copy files to start the game from disk.
    If you are using a different OS than the destination platform you may need 
    to modify the files or use the ones from the target OS system.
6-Prepare WHDLoad package if you want to distribute your game for high-end 
  AMIGAs, emulators or A500 mini.


It can also be done via CLI, this way you can have a script to build your
project. 

First parameter is the path to the project.
(Optional) Second parameter is the target platform (GENERIC/A500/A1200)
, GENERIC by default.
(Optional) Third parameter specifies if we want to pack resources,
(PACK/NOPACK), default is NOPACK.
(Optional) Last parameter can be used to make WHDLOAD package, by default 
is no package. 

Examples

    RedPillEditor020 projects/mirror/game.pill 
    RedPillEditor projects/mirror/game.pill A500
    RedPillEditor projects/mirror/game.pill GENERIC
    RedPillEditor projects/mirror/game.pill A500 PACK
    RedPillEditor projects/mirror/game.pill GENERIC NOPACK WHDLOAD
