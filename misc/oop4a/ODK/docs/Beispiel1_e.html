Example of an abstract class

abstract class Sort { long Root DataPointer protected makeRoot() { root=makeEntry() } abstract public makeEntry() abstract public insertItem(long value) abstract public deleteItem(long value) abstract public getFirstItem() abstract public getNextItem() abstract public displayList() } Class ChainedList extends Sort { public makeEntry() { long mem mem=new("ListEntry",0) return mem } public insertItem(long value) { Object Entry Entry=getLastItem(Root) Entry.AddEntry(value) } ... } Class BinaryTree extends Sort { public makeEntry() { long Entry Entry=new("BinTreeEntry",0) return Entry } public insertItem(long value) { Object Entry SortInTree(Root,Value) } ... } As you can see above, all derived objects have the same methods and methodearguments, but different methodebodies. A program could choose any of the derived classes at runtime and does not need to be changed. Any new sortclass could be used, even years after the original program was written and compiled.
