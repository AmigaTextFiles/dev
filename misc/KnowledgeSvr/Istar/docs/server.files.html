Istar Server Ancillary Files

For the Istar Knowledge Server to run properly, you must have the following files in the IstarSys:files/ directory (drawer). At present, assume all are mandatory, because quite a number of them hold HTML code and structure. In future I hope to rationalise them.

  • A set of files from which HTML pages are created:
    • htmlhead - HTML code to start a new page, up to title.
    • htmlend - Text to be added at the end of most pages; currently just a reference to Istar itself and the HTML ending code.
    • htmlwelcomestart - Starting text of Welcome Page - preceding the list of KBs.
    • htmlwelcomeend - Text to end Welcome Page, after the list of KBs.
    • htmlnotes - Notes to be added before the end of most pages; currently instructions on what to do on a question page.
    • htmlh0 - Heading for main help text of each question.
    • htmlh1..7 - Headings for subsidiary help texts 1..7
    • htmlhelppage - HTML page for Help that is not specific to any question.
    • htmlstoppage - HTML page to respond to user's 'stop' signal.
    • htmlwelcomedet - HTML text (e.g. H3 heading) that introduces the details of the KBs (long descriptions) on the welcome page.
    • htmlresults - Text to say that the inference session has stopped and results are available. Body text at start of the What-Now page.
  • The following contain error messages or pages (you MUST have these files):
    • htmlbad - Error message page saying that Istar did not understand what your browser sent; perhaps you had modified the URL?
    • htmlbadkbfile - Error message page saying that Istar could not load the KB file indicated in the KBDetails set up from the serverkbs file.
    • htmlbadkbid - Error message page saying that Istar does not have the KB indicated in the message sent from your browser; perhaps you had modified the URL, or were using an old Welcome Page that you had cached?
    • htmlfull - Error message telling client that Istar currently has its quota of clients.
    • htmlnogl - Error message page saying that the KB selected does not seem to have a Goal List; If this happens it means that the server KBs have been wrongly chosen, which is a fault at the server end.
    • htmlnokbc - Error message page saying that Istar cannot obtain a kbControl structure for the KB.
    • htmlnodoc - Body text to tell user that there is no document available.
