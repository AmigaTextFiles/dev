Системні вимоги

 

...для запуску програм, що встановлюються

  • Центральний процесор 68000 (рекомендується процесор не нижче 68010, тому що деякі slave-модулі вимагають VBR для функції виходу із програми по натисканню гарячої клавіші)
  • Kickstart 2.0 (версія не нижче 37)
  • Не менш 1 Мб оперативної пам'яті (іноді й більше, це залежить від установленої програми)
Емуляція Кікстарта

Деякі патчі використовують так званий kickemu для створення повноцінного середовища AmigaOS для встановленої програми. Для роботи kickemu потрібен образ оригінального кікстарта. Цей образ може бути отриманий шляхом зчитування кікстарта з реальної Аміги за допомогою утиліти GrabKick, яку можна знайти на сайтах aminet. Підтримувана версія образа кікстарта залежить від установленої програми. Інформацію про те який кікстарт повинен використовуватися, як правило, можна знайти в прикріпленому до патчу файлі ReadMe. Образи кикстартов версій 1.2 й 1.3 можуть бути отримані шляхом збереження їх з підходящих A500, кікстарт версії 3.1 може бути збережений з моделей A1200 або A4000. Якщо образ кікстарта не буде знайдений програмою WHDLoad або образ буде мати іншу версію або не буде повністю ідентичний оригінальному, то WHDLoad завершить роботу з відповідним повідомленням про помилку. Також мають бути присутні, відповідно конкретного образа кікстарта, додаткові файли переадресації з розширенням .RTB. Ці файли можуть бути знайдені на сайтах aminet в архіві util/boot/skick345.lha. Обидва файли, образ кікстарта й файл переадресації, повинні перебувати в директорії Devs:Kickstarts/ і повинні мати правильні назви. Наступна таблиця показує правильні назви файлів залежно від версії кікстарта:

1.3 Devs:Kickstarts/kick34005.A500 Devs:Kickstarts/kick34005.A500.RTB

...для створення нових патчей і програм інсталяції

  • рекомендується процесор не нижче 68030 з MMU для використання всіх можливостей WHDLoad (крута машина - завжди кращий вибір для розробки програмного забезпечення ;-))
  • знання машинних кодів, асемблера й аміжного заліза...
  • асемблер (добре, якщо що вміє оптимізувати)
  • інші програми (hexedit, disasm...)
  • трішки більше оперативної пам'яті
  • доступ до різних машин для можливості більш широкого тестування (68000 - 68060, OCS, AGA, GfxCard)
  • апаратний або програмний системний монітор типу AR3, HrtMon, TK
  • кнопка NMI для переривання програми, що виконується, або входу в налагоджувальник/системний монітор.
