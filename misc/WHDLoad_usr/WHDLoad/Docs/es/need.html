Requerimientos

... para ejecutar programas instalados

  • CPU 68000 (se recomienda 68010+ dado que algunos instaladores requieren el VBR para la funcionalidad 'salir pulsando una tecla')
  • Kickstart 2.0 (versión 37+)
  • un mínimo de 1.0 MB RAM (algunas veces mas, depende del programa instalado)
Emulación de la Kickstart

Algunos instaladores usan el llamado kickemu para disponer de un entorno AmigaOS completo para el programa instalado. Para usar dicho kickemu se requiere una imagen de la kickstart original. Esta imagen puede extraerse de una Amiga real con herramientas tales como GrabKick o similares que pueden encontrarse en aminet. La versión soportada de la imagen depende del programa instalado. Ud. puede encontrar información acerca de la imagen que debe usarse en el archivo ReadMe provisto. La imagen para versiones de kickstart 1.2 y 1.3 solo puede extraerse de una A500 apropiada, la versión 3.1 debe extraerse de una A500/A600/A2000, A1200 o A4000 la cual debe estar equipada con una ROM Kickstart 3.1. Si la imagen no puede ser encontrada por WHDLoad o la imagen es la versión incorrecta o está alterada en cualquier forma, WHDLoad terminará la ejecución con un mensaje de error apropiado. Adicionalmente, un fichero de reubicación con la extensión .RTB que se corresponde con la imagen de la kickstart debe estar presente. Este fichero de reubicación puede encontrarse en aminet en el archivo util/boot/skick346.lha. Ambos ficheros, la imagen de la kickstart y el fichero de reubicación deben estar ubicados en el directorio Devs:Kickstarts/ y deben tener el nombre correcto. Comenzando con WHDLoad versión 16.5 también pueden utilizarse las imágenes de kickstart del CD Amiga Forever de Cloanto. Las imágenes deben ser renombradas de acuerdo a la tabla debajo. ... para escribir nuevos Esclavos / programas instaladores

  • se recomienda un 68030+ con MMU para usar todas las funcionalidades de WHDLoad (una gran máquina es siempre una buena elección para desarrollar software ;-))
  • conocimiento de código máquina, ensamblador, hardware de amiga ...
  • un ensamblador (debe poder optimizar)
  • algunas otras herramientas resultaran útiles (editor hexadecimal, desensamblador, ...)
  • algunos MB's adicionales de RAM
  • acceso a diferentes máquinas para poder hacer pruebas (68000 - 68060, OCS, AGA, tarjeta gráfica)
  • un congelador por hardware o software tal como AR3, HrtMon, TK
  • un botón de NMI para interrumpir un programa en ejecución o para entrar en el monitor de software/congelador
