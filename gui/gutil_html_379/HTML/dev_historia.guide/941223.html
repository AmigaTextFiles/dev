 ~~~~~~~
	* Några av rutinerna skickade tillbaka resultatet i "fel" register.
	  Det gör inget om man bara använder libraryt från assembler, men
	  alla andra programmeringsspråk kräver att resultatet alltid kommer
	  tillbaka genom D0.

	  Några av rutinerna skickar nu tillbaka resultat i flera register
	  (ofta A0 och D0).

	* Ändrat så att GU_GadgetArrayIndex använder A0 som pekare till
	  LayoutGadget arrayen (internt används A1 som pekare).

	* Skrev FD filen och konverterade till Lattice/SAS C, Aztec, DICE
	  och Maxon C (pragma filer = motsvarar LVO för assembler).
