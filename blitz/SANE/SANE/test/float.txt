
For those who might be interested. :)

This is a simple comparative analysis of Amiga FFP vs Single IEEE float,
which I devised to help me understand the differences between the two
float types.

Calculations for both float types were made using Single IEEE float.

exp1  = RAW exponent value

exp2  = Calculated exponent value

frac1 = RAW fraction value

frac2 = Calculated fraction value


 Amiga FFP:-

2.66666674613952 = 8/3;  exp1=66  exp2=4   frac1=11184811 frac2=.666666686534881

3.14285707473754 = 22/7; exp1=66  exp2=4   frac1=13182098 frac2=.785714268684387

.666666686534881 = 2/3;  exp1=64  exp2=1   frac1=11184811 frac2=.666666686534881

.33333334326744  = 1/3;  exp1=63  exp2=.5  frac1=11184811 frac2=.666666686534881


 Single float:-

2.66666674613952 = 8/3;  exp1=128 exp2=2   frac1=2796203  frac2=.333333373069763

3.14285707473754 = 22/7; exp1=128 exp2=2   frac1=4793490  frac2=.571428537368774

.666666686534881 = 2/3;  exp1=126 exp2=.5  frac1=2796203  frac2=.333333373069763

.33333334326744  = 1/3;  exp1=125 exp2=.25 frac1=2796203  frac2=.333333373069763


From these results I determined that there was a difference of 62 in the
exp1 variables between the two float types.

I also determined that there was a difference of 8388608 in the frac1
variables between the two float types.

This information was the basis for me to go on and produce more 
float functions.
