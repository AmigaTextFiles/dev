68000 microprocessor introduced 1979 by motorola.
US american eletronics company, chicago.
continiuated as power reduced variant (15mA) by freescale semiconductor.

32bit addressing system, 24bit external address bus.
32 bit data operation, 16bit external bus.

8 32bit address registers, a0-a7 a7 is the stack pointer (usp)
8 32bit data regsisters, d0-d7
ccr 8bit condition code register (lower byte of sr)
pc 32bit program counter
sr 16bit status register
ssp 32bit supervisor stack pointer

the processor has two modes: user mode and supervisor mode.
some of the intructions are "priviledged", accessing them from user mode
causes an "exception". usually the core subroutines of the operating system
run in supervisor mode. the programs run in user mode. this allows
taskswitching/multitasking.

legend
effective addresscan be anything including: address register,
immmediate memory location, one of the addressing modes.
certain instructions are restricted that they are not allowed
all modes/types of registers. see the motorola reference for details.
data registerinternal 32bit data storage
data sizethe size of self-contained/accessed data.
address registerpoints into external memory.
data and address registers are interchangeable.
address mode (by size)16bit/32bit wide.
the 68000 does not implement the upper 8bits.
externally, addresses are always 24bit.
immediatea numeric value of 3,4,8,16 or 32bit size.
sectionphysical memory area. an assembler directive.
it is common to put data/code into different sections.
important address modes: (others see motorola reference)
(a0)address register indirect (ari)
offset(a0)ari 16 bit displacement
-(a0)ari pre-decrement (derived designs: not inlcluded)
(a0)+ari post-increment (derived designs: not included)
offset(pc)16bit offset to program counter
important common abbreviations:
d8/d168/16 bit distance (offset)
an/dnany of a0 to a7/d0 to d7
.b8bit data size (move.b)
.w16bit data size (move.w)
.l32bit data size (move.l)
one strategy is it not to spend time about the different addressing modes,
and use (a0) (ari) 32 bit mode exclusively. this brings with it a slight
performance loss, however, by good loop design such programs are the better
choice. an increase in code size is unavoidable. the offset (16bit) (pc)
(program counter) mode is also good, and easy to use.
this statement is an objection, and just suggestive to start coding more easy.
indeed adding to address registers allows to simulate the various addressing modes.
in example, the mode d8(dn,an) performs this: add "dn" register to "an" register.
add 8 bit distance to "an" register.

linklink and allocate.
sequence:
push address register onto stack.
load stack pointer into address register.
add displacement to address register.
syntax:address register/immediate displacement (16bit)
example:link a0,#128
the following instruction sequence performs the same.
push a0. mvusp a0. subi.l #128,a0
then, it is possible to apply the d16(an) addressing mode.
if addressed reverse, the "push a0" instruction decrements usp by four.
it is also possible to load a data register (with -128)
for the d8(an,dx) addressing mode. d8 means 8bit distance.
this lists variations, using the link instruction or subtractive addressing.
